# Security Policy

| Version | Supported |
| ------- | -------- |
| Latest release on `main` | Yes |
| Older major versions | Best effort |

## Reporting a vulnerability

Email **[security@psydata.org](mailto:security@psydata.org)** with:

- Descriptions and impact
- Steps to reproduce
- Affected repos and versions

We aim to acknowledge within **5 business days**. Please do not disclose publicly until we coordinate a fix.

## Safe Harbor

We support good-faith security research on systems we operate when you follow this policy.

> [!IMPORTANT]
> Individual repositories may have an updated **SECURITY.md**, which will apply alongside this one.
