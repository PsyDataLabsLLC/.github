# Contributing

## Branching

**Default Branch**: `main`
**Feature Branches:** `feature/short-description`
**Releases:** `release/` and, or `version/`
**Fixes**: `fix/short-description`

## Commits

- Use clear, imperative subject lines
- Reference issues: `Fix login redirect (#42)`

## Pull requests

1. Open a PR against `main`
2. Fill out the PR template checklist
3. Ensure CI passes
4. Request review from CODEOWNERS where applicable

## Code review

- Address feedback or explain defferals in comments
- Squah-merge preferred unless release branch policy says otherwise

> [!IMPORTANT]
> Individual repositories may have another CONTRIBUTING policy that goes alongside this one.
